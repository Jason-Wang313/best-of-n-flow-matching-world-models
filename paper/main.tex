\begin{abstract}
Rectified-flow trajectory generators make it cheap to draw many plausible futures before acting. This paper studies the audit problem created by that convenience: when a learned proxy selects an upper-tail generated trajectory, is the selected future actually better, or has the generator exposed off-manifold artifacts that the proxy overrates? We train conditional rectified-flow models for continuous point-mass trajectories and for action-sequence generation on Gymnasium classic-control benchmarks. The audit measures realized return, proxy-realized gap, training-manifold distance, selection bias, mode entropy, Euler-step consistency, and oracle headroom inside the same sampled candidate pool. In the controlled point-mass world, proxy-tail selection at \(N=16\) reduces mean true return from \(-64.03\) to \(-95.68\), while a feature-calibrated selector improves it to \(-41.57\). A separate Euler-step sweep shows this is not explained away by coarse flow integration. On held-out Gymnasium seeds with \(N=32\), CartPole exhibits strict proxy-tail harm and strict feature-calibrated repair; MountainCar and Acrobot act as harder stress cases where feature calibration reduces selected OOD distance and mean proxy-realized gap but only weakly improves executed utility. The claim is therefore intentionally scoped: this is a falsifiable tail-value audit for rectified-flow trajectory models, not a new general-purpose planner.
\end{abstract}

\section{Introduction}

Generative world models increasingly make planning look like conditional generation: sample futures, score them, and inspect the most promising trajectory before execution. Rectified-flow models are attractive in this role because a learned vector field can transport base noise into structured continuous trajectories with a small number of Euler steps \cite{lipman2023,liu2023}. The safety question is not only whether average samples look plausible. A planner or analyst often cares about the selected upper tail, and that tail can be where proxy errors concentrate.

This paper frames the problem as a \emph{tail-value audit}. Given a conditional rectified-flow trajectory generator and a proxy value model, we ask whether high-proxy futures remain near the training trajectory manifold, whether their true executed return improves with candidate budget, and whether a repair mechanism survives baselines that should expose false explanations. The audit is deliberately adversarial during development: proxy-tail failures are treated as diagnostic teachers, not as results to hide.

The final protocol is frozen before reporting. It fixes code, seeds, candidate budgets, tasks, selectors, Euler-step controls, metrics, and claim gates. The resulting paper is meant to read narrowly but cleanly: a shortcut proxy fails for a specific reason; a manifold-aware selector repairs that reason in the mechanism-isolation setting; standard-control stress tests show where the repair is strong, where it is weak, and how much oracle headroom remains.

The project is differentiated from other papers in this collection by its object of study. It is not about language-model answer selection, RSSM latent imagination, memory-augmented rollouts, Bayesian ensembles, differentiable physics, or diffusion policies. The unit under audit is a trajectory produced by integrating a rectified-flow vector field. The main question is whether the chosen upper tail is executable, or merely attractive to an incomplete proxy.

\section{Audit Protocol}

Let \(p_\theta(x \mid c)\) be a conditional rectified-flow trajectory generator for context \(c\). For each context, we sample a candidate pool \(\{x_i\}_{i=1}^N\) and score each candidate with a proxy \(\hat R(x_i,c)\). The proxy-tail selector chooses
\[
i_{\mathrm{tail}} = \arg\max_i \hat R(x_i,c).
\]
The feature-calibrated selector subtracts a training-manifold distance penalty,
\[
i_{\mathrm{cal}} = \arg\max_i \hat R(x_i,c) - \lambda u(x_i,c),
\]
where \(u\) is a standardized distance of trajectory or action features from training behavior. We compare it with the first sampled candidate, a flow-residual selector, and an oracle selector that chooses the best true-return candidate inside the same generated pool. The oracle is not deployable; it only tests whether better candidates existed but were missed by the proxy.

The audit reports true executed utility \(R\), proxy-realized gap \(\hat R-R\), feature-space OOD distance \(u\), selection bias, diversity or mode entropy, Euler-step residual, and oracle headroom. A selector is not credited as a repair unless it improves true return while reducing the proxy-realized gap and selected OOD distance under the frozen claim gate.

\paragraph{Calibration sanity check.}
If \(|R(x_i,c)-\hat R(x_i,c)| \le \beta u(x_i,c)\) for every candidate in a pool, then for any comparison candidate \(j\), the calibrated selector with \(\lambda=\beta\) satisfies
\[
R(x_{i_{\mathrm{cal}}},c) \ge R(x_j,c) - \beta u(x_j,c) - \beta u(x_{i_{\mathrm{cal}}},c).
\]
This is a standard robust-selection lens, not the paper's novelty. It clarifies why the penalty must be evaluated empirically: if \(u\) is poorly calibrated or misses the relevant proxy error, the inequality gives no useful practical guarantee.

The final claim gate has three layers. \section{Synthetic Rectified-Flow World}

The mechanism-isolation environment is a two-dimensional point mass moving from \((0,0)\) to a context-dependent goal while avoiding a circular obstacle. Each context specifies goal height, obstacle radius, and lateral wind. Expert trajectories use two modes around the obstacle, which makes diversity meaningful without requiring a large simulator.

The generator is trained as a conditional rectified flow. For data trajectory \(x_1\), Gaussian base sample \(x_0\), and interpolation time \(t\), the vector field predicts
\[
v_\theta((1-t)x_0 + t x_1, t, c) \approx x_1 - x_0.
\]
At inference, Euler integration maps Gaussian noise to complete candidate trajectories conditioned on \(c\). The proxy value model is a ridge regressor trained on endpoint and mode-height features only. It intentionally omits direct obstacle-collision, roughness, path-length, and backtracking terms. This makes the proxy accurate enough near the expert manifold but vulnerable to selected upper-tail artifacts.

\paragraph{Synthetic finding.}
At the largest budget, proxy-tail selection worsens mean true return by \(31.65\) relative to the first candidate and by \(54.11\) relative to the calibrated selector. \section{Gymnasium Control Stress Test}

Controlled evidence is not enough for a strong empirical claim, so v4 adds a standard benchmark tier using Gymnasium classic-control environments \cite{towers2024gymnasium}. We evaluate CartPole-v1, MountainCar-v0, and Acrobot-v1. For each task, the rectified-flow model generates full open-loop action sequences conditioned on the initial environment state. Generated continuous action values are discretized to legal task actions and executed in the real Gymnasium environment. The proxy deliberately overvalues a shortcut feature: sustained one-sided action plans that look simple to the proxy but can fail under the environment dynamics. Feature calibration uses task-specific action-shape statistics fixed by the development smoke pass; final reporting uses held-out seeds 70--74, six evaluation contexts per task and seed, and \(N \in \{1,2,4,8,16,32\}\).

\paragraph{Benchmark finding.}
The standard-control tier supports a more nuanced claim than the synthetic world. CartPole shows strict proxy-tail harm and strict feature-calibrated repair: the proxy tail loses \(5.78\) utility relative to the first candidate, while calibration gains \(9.81\) over the proxy tail and reduces both selected OOD and proxy-realized gap. MountainCar and Acrobot do not show strict utility repair. They still show positive oracle headroom, nonnegative mean calibration effects, lower selected OOD distance, and lower mean proxy-realized gap. The audit therefore discovers a real boundary of the mechanism rather than hiding it. The final claim gate requires CartPole to pass strictly, every Gymnasium task to retain positive oracle headroom, and residual controls to reduce step sensitivity on all three tasks.

\section{Failure Analysis}

\paragraph{Why CartPole is the cleanest standard benchmark.}
The shortcut proxy rewards sustained one-sided action sequences because they are simple in action-feature space and can appear high-value under a proxy that ignores closed-loop balance. Executed CartPole rollouts punish this shortcut quickly: one-sided action sequences lose pole angle margin and terminate early. The flow still samples better candidates, as shown by the large oracle-proxy gap, and the feature-calibrated selector recovers part of that missed value by rejecting high-OOD sustained actions.

\paragraph{Why MountainCar and Acrobot are harder.}
MountainCar and Acrobot both contain useful high-energy phases. A proxy that overvalues sustained action can therefore be wrong in some contexts and accidentally helpful in others. This is precisely why the paper does not claim a universal repair. The calibration score reduces selected OOD distance and mean proxy-realized gap, but the executed-utility intervals remain mixed. In reviewer terms, these tasks are negative controls against a too-neat story: the mechanism detects proxy-tail risk and headroom, but the current hand-built feature penalty is not enough to solve every benchmark.

\paragraph{What the oracle upper bound rules out.}
The oracle selector is not a deployable method, but it answers an important alternative explanation. If the proxy tail were poor because the flow never generated good candidates, oracle-proxy gaps would be near zero. Instead, every Gymnasium benchmark has positive oracle headroom. The failure is therefore at least partly a selection problem inside the sampled pool, not merely a generation-capacity problem.

\paragraph{What the residual control rules out.}
Rectified-flow samplers depend on numerical integration, so a skeptical reviewer could argue that tail failures are coarse-Euler artifacts. The residual control directly attacks that hypothesis. It lowers selected coarse-vs-fine residuals across the synthetic sweep and all Gymnasium benchmarks, but it is not the strongest true-return repair. Step sensitivity is a useful diagnostic, not the whole explanation.

\section{Implementation and Compute}

The implementation is deliberately CPU-friendly. The synthetic generator and the Gymnasium action generator both use small conditional vector-field networks trained with rectified-flow interpolation losses. The Gymnasium full benchmark trains five seed-specific action-sequence generators with 56 training contexts per task, 120 optimization steps per seed, six held-out evaluation contexts per task and seed, and at most 32 candidates per context. The largest final benchmark writes 2700 selector rows and 15 task-by-seed effect rows.

The feature penalty is task-scaled because reward and proxy-score units differ across Gymnasium tasks. These scales are part of the frozen protocol and are written into \texttt{results/gym\_control/aggregate.json}; the final held-out seed block is separate from the development smoke pass used to diagnose the selector. The paper therefore treats the feature penalty as an auditable baseline, not as a learned universal uncertainty estimator.

All final claims are generated from saved CSV/JSON artifacts rather than manual transcription. The claim audit fails closed if a required aggregate is missing, if seed counts are too small, if Gymnasium is not run in full mode, if CartPole loses strict repair, if any benchmark loses oracle headroom, or if residual controls stop reducing selected step residuals. This gives a future reviewer or fresh agent a concrete way to reproduce the stated scope.

\section{Related Work}

Flow Matching \cite{lipman2023} and Rectified Flow \cite{liu2023} provide the generative modeling foundation for this experiment. Stochastic interpolants and diffusion probability-flow views make clear that rectified-flow samplers share many behaviors with other continuous generative models; this paper therefore avoids claims of flow-specific uniqueness. World-model systems such as PlaNet, Dreamer, TD-MPC, EfficientZero, and DreamerV3 \cite{hafner2023} are broader control algorithms than the small audit here. Diffuser-style trajectory generation \cite{janner2022} is a close neighbor because it also treats behavior as conditional trajectory generation. Reward-model overoptimization \cite{gao2023} is the parent phenomenon for proxy-selected tails. Gymnasium supplies the standard environment interface and reference classic-control tasks used for the v4 stress benchmark \cite{towers2024gymnasium}. The contribution here is the rectified-flow trajectory audit: continuous paths or action sequences, candidate-pool diagnostics, held-out true returns, flow-step controls, and claim gates that distinguish strict repairs from stress-case boundaries.

\section{Limitations}

The experiment is intentionally small. The Gymnasium tier uses classic-control tasks and open-loop action sequences, not high-dimensional robotics, pixel observations, or offline-control datasets. The proxy is deliberately incomplete, not a learned high-capacity value model. The uncertainty score is hand-built feature distance, not an ensemble, conformal predictor, or learned uncertainty estimator. Feature calibration is strong in the mechanism-isolation world and in CartPole, but only weakly improves executed utility on MountainCar and Acrobot. The right interpretation is therefore narrow: this paper supplies an auditable failure case, a repair that works under scoped conditions, and a benchmark harness that makes its own boundaries visible.

\section{Conclusion}

Rectified-flow trajectory models can produce plausible average samples while their proxy-selected upper tail becomes less executable as candidate budget grows. A tail-value audit exposes this by measuring true return, proxy-realized gap, OOD distance, selection bias, diversity, Euler-step consistency, and oracle headroom on selected candidates. The controlled world shows the clean mechanism; the Gymnasium benchmark shows both a strict CartPole repair and harder stress cases where the repair is only partial. That is the intended contribution: a submission-facing audit protocol that makes proxy-tail failures, repairs, and boundaries measurable.

\section{Metric Definitions}

This appendix gives the concrete interpretation of each metric used in the claim gate. \emph{True return} is the held-out evaluator score for the selected candidate. In the synthetic world it combines endpoint accuracy, obstacle avoidance, smoothness, path length, and backtracking penalties. In the Gymnasium tier it is the executed rollout utility computed from real environment transitions and task-progress shaping. Higher is better.

\emph{Proxy-realized gap} is \(\hat R(x_i,c)-R(x_i,c)\) for the selected candidate. It is not an average prediction error over the candidate pool. It is the error on the action or trajectory that a tail selector would actually choose. A high gap therefore means the selected candidate looked better to the proxy than it was under execution.

\emph{Feature-space OOD distance} is the standardized distance between selected candidate features and training behavior. In the synthetic world, these features describe trajectory geometry, mode, roughness, and obstacle interaction. In the Gymnasium tier, they describe action-sequence shape, including mean action, energy, switch rate, maximum run length, roughness, segment means, lag-one action correlation, and early-late action shift. Lower selected OOD is not treated as sufficient for success; it must coincide with true-return improvement or at least nondegradation under the scoped Gymnasium gate.

\emph{Selection bias} is the selected proxy score minus the mean proxy score in the candidate pool. It measures how aggressively the selector chases the upper proxy tail as \(N\) grows. \emph{Mode entropy} is used in the synthetic two-mode obstacle world to test whether selection collapses onto one obstacle-avoidance mode. \emph{Flow residual} is the root mean square candidate difference between coarse and fine Euler integration of the same rectified-flow model. It is a numerical-consistency diagnostic, not the main uncertainty estimate.

\emph{Oracle headroom} is \(R(x_{\mathrm{oracle}},c)-R(x_{\mathrm{proxy}},c)\), where \(x_{\mathrm{oracle}}\) is the best true-return candidate inside the same generated pool. Oracle headroom is not a deployable result. It is included because it separates two failure modes: bad generation, where no good candidate exists, and bad selection, where the generated pool contains better candidates that the proxy ignores.

Two design choices are worth making explicit. First, the Gymnasium policies are open-loop action sequences rather than closed-loop controllers. This keeps the unit of analysis aligned with rectified-flow trajectory generation and makes candidate-pool selection directly auditable, but it also limits control performance. Second, the proxy is intentionally incomplete. The paper is not claiming that a modern learned value model would fail in the same way without evidence. It claims that once a proxy has a shortcut, upper-tail selection can amplify that shortcut, and the proposed audit can detect whether calibration actually repairs the selected candidate.

\section{Reproducibility Checklist}

The final PDF is considered reproducible only when the following checks pass in order. The repository must install with \texttt{python -m pip install -e .[dev]}. Unit tests must pass with \texttt{pytest}. The synthetic aggregate, step sweep, and Gymnasium benchmark must exist under \texttt{results/}. The paper figures must be regenerated under \texttt{paper/figures/}. \texttt{scripts/run\_claim\_audit.py} must return \texttt{"status": "pass"}. \texttt{scripts/build\_paper.py} must compile a v4 PDF without a LaTeX failure log. Finally, the rendered PDF must show all figures and wide tables without clipping or overlap.

The audit intentionally stores intermediate CSV files, not only summary numbers. This makes it possible to re-run alternative claim gates after review. For example, a reviewer can inspect \texttt{results/gym\_control/\allowbreak metrics.csv} to compare selectors at \(N=8\) or \(N=16\), or inspect \texttt{results/gym\_control/\allowbreak effects.csv} to see which task-seed pairs produce weak calibration effects. The paper's claims use only the frozen \(N=32\) aggregate, but the row-level data are retained to make hidden cherry-picking harder.
